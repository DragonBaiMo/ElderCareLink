# 答辩材料（关爱空巢老人管理系统）

## 系统简介
- 面向社区空巢老人关爱场景，支持老人档案、健康记录、探访记录、公告发布与统计。
- 单体架构，前后端分离，强调“能跑即可”的毕业设计目标。

## 架构图（ASCII）
```
[Vue3+Element] --Axios--> [SpringBoot Controller] --Service--> [JPA Repository] --> [MySQL]
                                   ^                                    |
                                   |--- JWT Token & 校验 ---------------|
```

## 主要流程图（登录与业务操作）
```
用户 -> 登录页 -> /api/auth/login -> 返回JWT -> 携带Token访问各业务接口 -> 前端展示列表/表单
```

## 数据库结构（核心表）
- t_user：用户与角色（管理员/志愿者/家属）
- t_elder：老人基础信息
- t_health_record：健康指标记录
- t_visit_record：志愿者探访记录
- t_announcement：公告信息
- t_dict：数据字典（居住类型、健康等级）

## 技术亮点
- JWT 轻量认证，便于前后端分离演示。
- 数据初始化脚本预置账号与示例数据，开箱即演示。
- 统一响应与全局异常处理，界面提示均为中文，符合毕业答辩易读性。
- Vue3 + Element Plus 快速搭建表格/表单，提高开发效率。

## 开发总结
1. 按 PDR/PRD/IPO 拆解需求，优先保证 Must 级功能闭环：登录 → 建档 → 录健康/探访 → 公告 → 统计。
2. 采用经典单体分层：Controller-Service-Repository，便于答辩阐述。
3. 数据库设计围绕“用户-老人-健康-探访-公告”，字段简洁，满足查询与扩展需求。
4. 前端以表格+弹窗的常见后台风格实现，降低学习成本，保证演示直观性。
